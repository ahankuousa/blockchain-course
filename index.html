第二章

區塊的結構

每個區塊就像一頁帳本，記錄著重要的交易資訊

區塊 #1 2026-04-03 10:00:00
e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855

📚 知識點

  • Index：區塊在鏈中的位置，從 0 開始
  • Data：區塊中儲存的資料（如交易記錄）
  • Previous Hash：連接前一區塊的「鎖鏈」
  • Nonce：挖礦時變化的數字，用於產生符合條件的 Hash
  • Hash：區塊的數位指紋，任何資料改變都會讓 Hash 完全改變
第三章

Hash 計算

試著改變輸入，觀察 Hash 如何戲劇性變化！

9f86d081884c7d659a2feaa0c55ad015a3bf4f1b2b0b822cd15d6c15b0f00a08
計算中...
字元數： 18
Hash 前綴： 9f86...

📚 知識點

  • 不可預測：無法從 Hash 反推原始資料
  • 確定性：同樣輸入永遠產生同樣 Hash
  • 雪崩效應：輸入改變一個字元，Hash 完全改變
  • 唯一性：不同資料幾乎不可能產生相同 Hash

⚠️ 篡改測試區

嘗試修改過去區塊的資料：

📚 知識點

  • 創世區塊：區塊鏈的第一個區塊，Previous Hash 為 0
  • 鏈接原理：每個區塊的 Previous Hash 指向前一區塊
  • 防篡改：改變任何區塊會讓後面所有 Hash 失效
  • 共識必要性：需要共識機制防止恶意篡改
第五章

工作量證明 (PoW)

點擊「挖礦」，體驗找到符合條件 Hash 的過程

3 個零
嘗試次數： 0
當前 Hash： -
目標： 000

📚 知識點

  • 礦工工作：不斷改變 Nonce，找出符合難度的 Hash
  • 難度調整：比特幣每 2016 個區塊調整一次，維持 10 分鐘/區塊
  • 51% 攻擊：若掌控超過 50% 算力，可逆轉交易
  • 能源消耗：PoW 的主要批評，導致其他共識機制的發展

📚 知識點

  • 分叉：兩個礦工同時挖出區塊，區塊鏈暫時分叉
  • 最長鏈原則：網路最終採用最長的鏈作為主鏈
  • 孤立區塊：較短鏈上的區塊會被拋棄（Orphan Block）
  • 六個確認：比特幣通常等待 6 個區塊確認認為交易安全

🚨 篡改測試

嘗試修改簽名後的訊息：

📚 知識點

  • 私鑰：只有擁有者知道，用於簽名交易
  • 公鑰：由私鑰推導，可公開，用於驗證簽名
  • 錢包地址：公鑰的 Hash，類似銀行帳號
  • 數位簽名：用私鑰對訊息簽名，任何人可用公鑰驗證
  • 不可否認性：簽名後無法否認是你簽的
